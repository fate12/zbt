# 数据库迁移文件

服务启动时会**自动**按文件名顺序执行本目录下尚未应用的 `.sql` 文件，
并用 `_migrations` 表记录已执行的文件。已执行的不会重复跑。

## 日常改字段 / 加表 / 加索引

1. 在本目录新增一个**编号比现有最大号大**的 `.sql` 文件，例如：
   ```
   002_add_anchor_phone.sql
   ```
2. 内容写 DDL（建议幂等，`IF NOT EXISTS`，便于重跑）：
   ```sql
   ALTER TABLE anchor_accounts ADD COLUMN IF NOT EXISTS phone VARCHAR(32) DEFAULT '';
   CREATE INDEX IF NOT EXISTS idx_xxx ON ...;
   ```
3. 提交、推送 → CI/CD 部署 → 服务重启即自动执行。**不用手动登数据库。**

## 规则

- 文件名：`数字_说明.sql`，数字决定执行顺序（`001` < `002` < `010`，别用 `2` 和 `10` 混排，会按字符串排序出错）。
- **不要修改已上线过的文件**——它们不会再执行，改了也没用。要改就加新文件。
- 单个文件内的多条语句会在**同一个事务**里执行，失败整体回滚、并中断服务启动（迫使修复，避免带着不一致的 schema 运行）。
- 含 `$$ ... $$` 的函数体可直接整段写，无需特殊分隔。

## 首次部署

`001_init.sql` 是初始全量建表脚本（从旧 `supabase/init-supabase.sql` 精简而来，
去掉 pgvector / RLS，仅含运行时表）。新空库启动后会自动跑这一个文件。
